# 🗡 Roguelike / dungeon run

A procedural dungeon descent: every run is unique because rooms are generated by `rand`, not by a writer.

## What the example does

The hero descends from the "dungeon gate" deeper and deeper, room after room. Each room is a random type (monster, chest, potion, trap, or empty), and between rooms sits a risk choice: go deeper for more loot, drink a potion, or leave for the surface alive. The player accumulates depth, gold, and potions, balancing greed against health. Death (`hp <= 0`) cuts the run short without keeping the loot — that's the "one more run" hook.

## The heart of the genre: a room-generator loop

The whole gameplay hangs on one loop label. `:descend` increments depth and rolls the room-type die:

```
:descend
depth = depth + 1
// Random room type.
room = rand(1, 6)
bg /content/bg/dungeon_room.jpg
if room <= 2 -> r_monster
if room <= 3 -> r_treasure
if room <= 4 -> r_potion
if room <= 5 -> r_trap
-> r_empty
```

The `if room <= N -> ...` cascade is a probability distribution: a monster comes up on 1–2 (most often), a chest on 3, a potion on 4, a trap on 5, and anything no `if` catches falls through to `-> r_empty`. After any room, control converges on `:after_room`, which offers the risk choice "deeper / drink a potion / leave". And death watches over every hit — `if hp <= 0 -> dead` instantly breaks the loop.

## Engine features used here

- **Reactive HUD** — a single `text` line with interpolation updates itself whenever variables change:
  ```
  text hud x=3 y=8 size=42 color=#d8c9f0 «Depth {depth}  ♥{hp}/{maxhp}  💰{gold}  🧪{potions}»
  ```
- **Generator loop** — the `:descend` label with a `-> descend` jump from `:after_room` drives an endless room loop.
- **`rand` for everything procedural** — room type `room = rand(1, 6)`, creature health `mhp = depth * 3 + rand(0, 4)`, damage `hit = rand(2, 6)`, loot `loot = rand(8, 20)`.
- **`chance` for luck-based forks** — `if chance(0.5) -> sneak_ok` decides whether you managed to slip past.
- **Combat loop** — the `:fight` label jumps back to itself via `-> fight` until someone drops.
- **Risk choice** — three options in `:after_room` with different consequences.
- **`expr=` on an option** — the potion is unavailable when you have none: `- 🧪 Drink a potion -> drink expr="potions >= 1"`.
- **`min` for capped healing** — `hp = min(maxhp, hp + 6)` never overheals past the maximum.

## Step-by-step breakdown

**Run stats.** All state variables are declared in the header — that's the entire "character":

```
hp = 12
maxhp = 12
gold = 0
depth = 0
potions = 0
```

**`:descend` — room generation.** Depth grows by 1, `room = rand(1, 6)` is rolled, and the `if` cascade routes to one of five room labels (see above). This is the only branching point — everything else flows back here.

**Monster room (combat with a counterattack).** The creature's strength scales with depth: `mhp = depth * 3 + rand(0, 4)`. The player gets to choose between fighting and slipping past:

```
:fight
hit = rand(2, 6)
mhp = mhp - hit
You strike for {hit}.
if mhp <= 0 -> mon_dead
bite = rand(1, 4)
hp = hp - bite
The creature snaps back for {bite}. (♥{hp})
if hp <= 0 -> dead
-> fight
```

The `-> fight` loop alternates the player's hit and the creature's counterattack until `mhp <= 0` (victory → `:mon_dead`, awards `drop = rand(3, 9)` gold) or `hp <= 0` (death → `:dead`). The alternative is `:sneak`: `if chance(0.5) -> sneak_ok`, otherwise the creature bites you on the way out.

**The other rooms.** They're short and go straight to `:after_room`: `:r_treasure` adds `loot = rand(8, 20)` gold, `:r_potion` does `potions = potions + 1`, `:r_trap` hits for `dmg = rand(2, 5)` (with an `if hp <= 0 -> dead` check), `:r_empty` is just flavor text with no consequences.

**`:after_room` — the risk choice.** The heart of the tension:

```
:after_room
- ⬇ Descend deeper (risky) -> descend
- 🧪 Drink a potion -> drink expr="potions >= 1"
- 🪜 Leave with the loot -> escape
```

"Deeper" returns to the generator (the risk grows), "potion" heals and shows the menu again (`:drink` → `-> after_room`), "leave" ends the run with the loot.

**Endings.** `:escape` — survival with trophies (`Depth {depth}, {gold} gold`), `:dead` — death with `fade to="black"`. Both lead to `-> __end`.

## Why this is "one more run"

Procedurality here isn't hand-drawn, it's computed: `rand` decides the room type, enemy strength, loot size, and damage. Replayability comes from every pass of the `:descend` loop generating a fresh room on the fly — there's no fixed script, only rules and dice. No two runs are ever the same, and the outcome is determined by the interplay of luck and the player's "get greedy or get out" decisions.

## Make it your own

- **New room types.** Widen the roll to `rand(1, 8)` and add `if room <= N -> ...` branches — a forge, a merchant, a stairway fork, a buff shrine.
- **Items and relics via inventory.** Declare `inv = []` and collect finds with `inv = push(inv, "amulet")`, then check effects with `has(inv, "amulet")`.
- **Difficulty scaling with depth.** The hook is already there — `mhp = depth * 3 + rand(0, 4)`. Trap damage and loot can scale the same way.
- **A mini-boss every N floors.** In `:descend`, add `if depth >= 5 -> r_boss` (or a divisibility check) before the normal room roll.
- **Meta-progression between runs.** With `save`/`load` and variables, carry things like unlocked relics or upgrade currency from run to run.
