## 2. По модулям — что тянет вниз / что отлично

1. **Ядро рантайма (89).** Отлично: чистый шов `ILvnStage` — интерпретатор без зависимости от Unity, гоняется headless; плотное покрытие (flow, выражения, откат/резюм, интерполяция); злонамеренное выражение гейтится в false, а не роняет рантайм. Вниз: `LvnPlayer.cs` 934 строки монолитен; «malformed content never crashes» протекает в неогороженных Newtonsoft-кастах и неограниченной рекурсии парсера выражений.

2. **UI / стейдж (86).** Отлично: `VnStage` — `sealed partial` с чистым switch-диспетчером, не god-object; настоящие пустые состояния, лоадеры, offline-пути; 55+ тест-файлов. Вниз: `ResetStage` — «мягкий» сброс без отмены in-flight async → прошлая глава протекает в новую (актёры/фон/музыка/дубли спайна); сквозной «немой catch→null».

3. **Spine / анимации (89).** Отлично: самая отполированная подсистема — off-main-thread парсинг, MRU-кэш живых скелетов, warm-pulse против first-show хитча, `[lvn-perf]`-инструментирование. Вниз: LRU-эвикция ContentLoader может убить текстуры живого скелета (чёрные скелеты в длинной сессии); зависимость от reflection-seam `InitializeWithData`.

4. **Контент-пайплайн клиента (87).** Отлично: сетевой слой уровня прод-CDN-клиента — backoff, докачка через `.part`, версионный cache-busting, offline-сайдкары. Вниз: `ContentLoader.cs` 1654 строки перегружен; несколько TOCTOU в offline-стейт-сторе.

5. **Клиентские сервисы (80) — самый слабый из клиентских.** Отлично: кошелёк offline-first с server-authoritative ledger, kill-safe очередь, защита при смене аккаунта. Вниз: `static`+PlayerPrefs усложняют DI/мокинг; ads/IAP/analytics/daily/leaderboard без прямых unit-тестов; нет ретраев в `LvnBackend`.

6. **Go-сервер: контент+статика (86).** Отлично: atomic-write через temp+rename везде, per-path мьютексы, защита от zip-бомб. Вниз: нет `fsync` (атомарно против краша, не против потери питания); `filepath.Walk` в export глотает ошибки.

7. **Go-сервер: сервисы (85).** Отлично: сейвы на TOFU-ключе + OCC (409), Apple JWKS с пиннутым RS256, IAP replay-guard по txID, constant-time сравнения. Вниз: `/v1/wallet/earn` открыт (тест-режим до подключения платежей — помечено `SECURITY-TODO`); replay-guard IAP только внутри одного кошелька; безлимитная регистрация.

8. **Админка (70) — самый слабый модуль.** Отлично: whitelisted-конфиги, история версий с rollback, XSS-эскейп, guard двойного клика. Вниз: монолит 600 строк vanilla-JS; рендер-функции вкладок без try/catch → тихий пустой экран; `dirty=false` до подтверждения записи.

9. **IDE-панель (74).** Отлично: уровень настоящей IDE — автосейв драфтов, QuickOpen, ghost-подсказки, Problems dock, beforeunload-гард. Вниз: ~700 строк мёртвого кастомного `Editor` после миграции на Monaco; нет тестов/линта; удаление глав/тайтлов без confirm.

10. **Компилятор lvnconv (87).** Отлично: цельная многоформатная тула (CLI+wasm+MCP), стабильные id + i18n, 129 зелёных тестов, границы импортёров с номером строки и Левенштейн-подсказками. Вниз: «худший класс» — молчаливая перекомпиляция авторского текста не в то — открыт с двух сторон (известные опы с кривыми параметрами → реплика; `text`/`voice`/`scene`/`Слово =` → съедают прозу); позиции ошибок валидатора по индексу команды, не по строке `.lvns`.

11. **Веб-плеер /play/ (79).** Отлично: компиляция тем же `lvns.wasm`, что CLI; нарративное ядро полное; экспорт в один HTML; share-ссылки. Вниз: задеплоенный `lvns.wasm` устарел на 3 КБ; кнопка «Поделиться» была сломана (затенение `window.history` — починено); нет тестов/тач-жестов.

12. **Документация (84).** Отлично: двухслойная (howto для людей+агентов, docs-сайт) + `llms.txt`; все 12 жанров с рабочими `.lvns`. Вниз: CI компилирует только `hello.ink`, не все 12 примеров; разнобой RU/EN.

13. **Тесты / CI (87).** Отлично: многоязычный pipeline (Go×2, Node+grammar, panel-build, Unity EditMode+PlayMode, Pages, APK). Вниз: Unity/APK на единственном self-hosted mac (bus factor); smoke гоняет один пример.

## 4. Топ-5 инвестиций с максимальным подъёмом

1. **Вычистить мёртвый код и покрыть панель тестами/линтом** (IDE-панель 74, вес 10%): удалить ~700 строк мёртвого `Editor`, разбить монолит 1547 строк, подключить eslint + минимальные тесты. Наибольший вклад в взвешенную оценку.
2. **Durability денег на сервере** (Go-сервисы, надёжность 79): `f.Sync()` файла+директории в `atomicWrite`; не глотать ошибки persist в кошельке/auth (500, как state); per-user мьютекс вместо единого. Снимает единственный реальный риск потери денег.
3. **Оживить админку** (70 — минимум): try/catch с error-toast в рендер-функциях вкладок, loading/empty/error-состояния. ~день, надёжность 62→~80.
4. **Синхронизировать деплой веб-плеера + добить smoke** (/play/ 79, CI): передеплоить устаревший `lvns.wasm`; compile-проверка всех 12 жанровых `.lvns` в CI. Закрывает дрейф артефакта и гниение примеров.
5. **Снизить bus factor CI и позиции ошибок валидатора** (CI, lvnconv): продублировать/задокументировать Unity/APK-джобы; в валидаторе перейти с индекса команды на исходную строку `.lvns`.

Фундамент (ядро, компилятор, контент-слой, Spine, CI) — 86–89% и почти без техдолга; дельта-потенциал сконцентрирован в продуктовой обвязке. Пять инвестиций — «дни, не недели» — поднимают взвешенную готовность с ~84% до ~89–90%.
