# Что «выстреливает» на GitHub — и как перенести это на LVN (Unity)

Разбор ~30 успешных репозиториев, вывод общей закономерности и **конкретный план
переноса её на LVN — строго под Unity** (рантайм только Unity; в браузере не
играем — и это сознательно не нужно).

> **Update 2026-07:** тезис «в браузере не играем» пересмотрен жизнью — с 4 июля
> существует браузерный playground `/play/` (тот же wasm-компилятор, story-core,
> share-ссылки, экспорт в один HTML), и именно он стал верхом воронки: «попробуй
> за 10 секунд без установки» → Unity остаётся способом **шиппить** игру
> (см. `docs/playground.md` и `llms.txt`). Продуктовый вывод ниже — «главный
> рантайм и цель — Unity» — в силе; изменился только acquisition-канал.

## 2. Общая закономерность (повторяется у всех)

1. **Первый успех за < 5 минут.** Для Unity это «добавил пакет → открыл sample-сцену → нажал Play».
2. **Одна острая фраза-позиционирование** («X для Y» / против инкумбента).
3. **README demo-first** — gif/скриншот работающего продукта в первом экране.
4. **Галерея запускаемых примеров / шоукейс.**
5. **Низкий порог (no-code/near), высокий потолок.**
6. **Встроенный шеринг артефактов** (для нас — шеринг `.lvns`-сценариев, не браузер-игр).
7. **Доки как продукт.** 8. **Оседлать волну.** 9. **Drop-in / ноль трения.**
10. **Сигналы доверия:** MIT, CI, релизы, CHANGELOG, semver, лого.
11. **Стартер-шаблон / sample-проект.** 12. **Комьюнити.** 13. **Пруф шипнутыми играми.**

## 4. План переноса на LVN (Unity-нативно)

### Главный рычаг (наша уникальная комбинация)
Не браузер. Для строго-Unity продукта побеждает **drop-in + мгновенный Play +
авторство без кода**, помноженное на AI-волну:

> **Unity-дев добавляет пакет → открывает sample-сцену → жмёт Play → полноценная
> игра идёт. Дальше сценарист (или LLM) пишет `.lvns`, кидает в `Assets/` — оно
> само компилируется и играет в его игре.**

Никто из соседей не делает ставку на **AI-авторство как первый класс** для
Unity. Позиционирование:
**«ffmpeg for narrative games — a Unity engine whose scripts an LLM can write.»**

### P0 — дни, максимальный рычаг
- [ ] **`.lvns` ScriptedImporter** в Unity-пакете: бросил `.lvns` в `Assets/` →
  Unity сам зовёт транскодер и даёт играбельный `.lvn`-ассет. Это **убирает
  внешний CLI-шаг** и делает «первый успех» Unity-нативным (как `.yarn` у Yarn
  Spinner). Самая высокорычажная фича. (#1,#9)
- [ ] **UPM Samples~**: готовая sample-сцена + sample-история, чтобы
  «Add package → Import Sample → Play» занимало < 5 минут. (#1,#4)
- [ ] **README demo-first**: в самый верх gif игры в **Unity Game view** + острый
  подзаголовок + блок «Install & Play in 5 min» (git URL). (#2,#3)
- [ ] **GitHub Release `v0.4`**: `.unitypackage` + бинари `lvnconv`
  (mac/win/linux) + корневой `CHANGELOG.md` + баннер/лого. (#10)

### P1 — недели
- [ ] **Дистрибуция, где ищут Unity-девы:** публикация в **OpenUPM** (бейдж в
  README) и листинг в **Unity Asset Store** (главный канал открытия). (#10)
- [ ] **package.json до Asset-Store-уровня:** `displayName`, `description`,
  `documentationUrl`, `changelogUrl`, `samples`, semver. (#10)
- [ ] **Шоукейс:** `howto/` 12 жанров → демо-сцены/скриншоты прямо в Unity. (#4)
- [ ] **Пруф:** собрать и опубликовать 1–2 мини-игры (itch.io standalone-сборка
  из Unity) как референс + гайд. (#13)
- [ ] **Комьюнити-вход:** Discord/Discussions, good-first-issue, видный
  `CONTRIBUTING`. (#12)

### P2 — экосистема
- [ ] **Редакторное окно LVN в Unity:** компиляция/превью `.lvns` без выхода из
  редактора (как Inky/Yarn) — добивает DX. 
- [ ] **AI-волна как маркетинг:** демо «LLM пишет нарративную игру в Unity за
  один промпт» + prompt-spec (уже есть). (#8)
- [ ] **Плагин-экосистема:** фронт-энды (форматы) и эффекты как модули —
  архитектура `.lvn` уже позволяет; оформить точку расширения.

### Метрика успеха
Не звёзды, а **drop-in адопция**: сколько Unity-девов прошли путь «увидел README
→ добавил пакет → sample-сцена играет → бросил свой `.lvns` и он заработал в их
проекте». Весь P0 существует, чтобы этот путь был коротким и не выходил из Unity.

## 5. Что НЕ делаем (сознательно)
- ❌ Браузерная игра как продукт и «шеринг игры по URL» — мы строго под Unity.
  Веб-панель/playground остаются лишь **инструментом авторинга/доками**, не
  каналом игры.
