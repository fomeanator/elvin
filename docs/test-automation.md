# Лестница полной автоматизации тестирования

Стоячая цель (задана 2026-07-19): каждые ~5 минут делается ОДИН шаг к состоянию
«всё тестируется само»: от чистого плеера до бота, который сам тыкает APK.
Правила тиков: один шаг за тик; чекбокс закрывается только когда шаг реально
работает (прогон зелёный); коммит после каждого шага; редактор Unity закрыт;
дев-сервер юзера на :8078 не трогать (тестовый порт — :8099); партнёрский
контент (scripts/cold-*, art/, sprites/cold*) в git не попадает; прод — только
чтение. Полный EditMode-прогон ~3 мин: гонять через `-testFilter` пока идёт
разработка шага, полный — на закрытии шага.

Прогон EditMode: `Unity -batchmode -projectPath sandbox -runTests
-testPlatform EditMode -testResults r.xml -testFilter "<Fixture>"`.

- [x] D1. Быстрая версия: `qa/monkey.sh` — adb monkey по com.ominis.timeromance

      Сделано 2026-07-19: скрипт поднимает headless AVD, ставит APK, ждёт
      маркер `[lvn-boot] … veil handed off`, скриншотит, гоняет monkey,
      грепает крэш-сигнатуры; отчёты в qa/reports/ (в .gitignore).
      Прогон зелёный: бут 5.5с, 500 событий, monkey сам вошёл в главу.
      Грабли (вшиты в скрипт): (1) дефолтный для -no-window SwiftShader вешает
      Unity 6 до первого лога — нужен `-gpu host`; (2) с Vulkan-фичей эмулятора
      Unity 6 (Auto API) виснет на инициализации — нужен `-feature -Vulkan`
      (движок падает на GLES3); (3) оверлей «Viewing full screen» крадёт фокус
      → APP_CMD_LOST_FOCUS → Unity паузит цикл до первой сцены навсегда —
      лечится `settings put secure immersive_mode_confirmations confirmed`.
